## FAQ

**1) What’s the right way to run a vpn service speed test?**  
Test baseline first (no VPN), then test with VPN enabled on the same network, same device, same test server, same time window.

**2) What should I track besides download speed?**  
Ping and jitter. A VPN can look “fast” on throughput but still feel bad in calls and games if jitter is high.

**3) Does encryption slow things down?**  
Some overhead is normal, but a good setup with **AES-256 encryption** and well-peered servers should still deliver high speed and a stable connection.

**4) Why enable Kill Switch during tests?**  
Because speed numbers don’t matter if your connection silently falls back to your ISP during a drop. The **Kill Switch** keeps the test honest.
